# 云谱 - 设计文档

## 项目介绍
云谱是一个自托管的中文个人CRM系统，用于管理人物、事件、提醒和家谱。

## 技术栈
- 前端：Vue.js + ElementPlus
- 后端：FastAPI + SQLAlchemy
- 数据库：SQLite（预留多数据库支持）
- 部署：Docker Compose

## 核心设计

### 关于初始化
项目有且只有一个初始化用户，用于创建第一个账号。该用户仅用于登录验证（保护隐私，防止他人访问），与系统内容无关。

初始化时自动创建一个人物，该人物代表使用者本人：
- 昵称字段值为"我"
- `is_me` 字段标记为 `true`
- 其他字段为空，用户可自行完善

### 关于仪表盘（原个人主页）
仪表盘作为系统首页，展示整体概览：
- 显示人物总数统计
- 显示所有人物的事件时间轴（按时间排序）
- 显示最近30天的日期提醒

#### 事件时间轴
事件时间轴展示所有人物的事件，按时间排序。
每个事件显示事件类型、事件日期、事件地点、事件描述等。
事件类型支持用户自定义。

#### 日期提醒
日期提醒展示所有日期提醒，按日期排序。
每个日期提醒显示提醒类型、提醒日期、提醒内容等。
提醒类型包括生日、纪念日、重要事件等。

### 关于人物详情页
展示人物的详细信息，右侧显示头像和基本信息，左上显示日期提醒，左侧剩余空间显示该人物的事件时间轴。

#### 人物信息
人物基础信息包含：姓、名、昵称、生日、逝世日期（NULL表示在世）、国家、故乡、现居地。

其他信息通过自定义字段存储，采用 JSON 格式：
```json
{
  "手机号": {
    "工作": "123****",
    "私人": "135****"
  },
  "邮箱": {
    "QQ邮箱": "asdaf@qq.com"
  }
}
```

关系设置：父母、子女，以便在家谱中展示。
新增关系时，选择关系类型：亲、干、义、养、继。
显示父母、子女的超链接，点击后跳转到该人物详情页。

### 关于家谱
家谱展示用户的所有人物，按血缘关系层次展示。

#### 展示规则
- 只连接直接关系（父母→子女），不展示间接关系连线
- 所有人物都显示（包括叔伯姑舅姨等）
- 例如：大伯连接到爷爷，与父亲处于同级

#### 视觉标识
- 亲生关系用实线表示
- 非亲生关系（干、义、养、继）用虚线表示
- 逝世者用灰色圆点标记，鼠标悬停显示"已故"

#### 关系计算
关系计算根据用户添加的关系，递归计算出所有人物相对于"我"的关系。

关系计算规则：
- **祖父母**：通过父/母的父/母计算得出
- **孙子女**：通过子/女的子/女计算得出
- **曾祖父母/玄孙**：递归计算多层级关系
- **兄弟姐妹**：拥有共同父母的成员，根据性别和出生日期判断哥哥姐姐还是弟弟妹妹
- **叔伯姑舅姨**：父母的兄弟姐妹

关系计算的起点是 `is_me = true` 的人物。

#### 关系存储
设计专用表存储计算后的关系（相对于"我"的视角）。
在新增或修改关系时，自动重新计算并同步更新该表。

### 关于提醒机制
- 仅实现站内通知
- 使用定时任务，每日24点检查并生成通知
- 支持周期性提醒（每年/每月/每周重复）
- 支持农历日期
