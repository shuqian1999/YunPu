# 个人关系管理系统（Personal CRM）详细设计文档

---

## 1. 引言

### 1.1 项目背景
现代人社交网络日益复杂，亲属、朋友、同事、客户等关系难以全面维护。商业 CRM 聚焦销售，不适合个人生活场景。本系统旨在打造一个**以人际关系质量为核心**的个人关系管理工具，帮助用户记住重要的人、保持联系频率、理清家族纽带。

### 1.2 目标用户
- 希望系统维护亲友信息的个人用户
- 需要记录互动历史与承诺事项的社交活跃人群
- 拥有庞大亲属网络、需管理称呼与家族树的用户
- 愿意通过绑定协作与亲友共享部分资料的小团体
- 中文母语人群

### 1.3 核心价值
- **记录**：完整保存联系人的基本信息、互动日志、自定义属性
- **关心**：生日、回访提醒，联系频率预警
- **连接**：绑定真实用户，信息自动同步
- **记忆**：时间轴浏览交往历史，家族树可视化亲属关系

## 2. 需求分析

### 2.1 功能需求
| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F1 | 用户注册、登录、个人资料管理 | P0 |
| F2 | 联系人增删改查，支持按分组/标签/国家/搜索筛选 | P0 |
| F3 | 自定义多层级分组和标签 | P0 |
| F4 | 记录与联系人的互动（时间、类型、摘要、心情） | P0 |
| F5 | 生日、纪念日、回访提醒，支持周期重复 | P0 |
| F6 | 联系频率设定与超期未联系提醒 | P1 |
| F7 | 自定义联系人字段（JSON扩展） | P1 |
| F8 | 亲属关系记录（系别、辈分、称呼） | P1 |
| F9 | 家族树可视化，支持展开/折叠 | P2 |
| F10 | 称呼自动计算引擎（支持多语言） | P2 |
| F11 | 联系人绑定系统用户，信息同步 | P1 |
| F12 | 绑定申请审批流程与解绑 | P1 |
| F13 | 系统内通知中心 | P1 |
| F14 | 管理员用户管理（禁用/启用/删除） | P1 |
| F15 | 管理员通用标签库、分组模板管理 | P2 |
| F16 | 系统公告发布 | P2 |

### 2.2 非功能需求
- **数据隔离**：所有用户数据严格按 `owner_id` 隔离，仅绑定字段跨用户同步
- **性能**：联系人列表加载 < 500ms，家族树查询 < 1s
- **安全**：密码 bcrypt 哈希，JWT 认证，管理员操作二次验证
- **可用性**：响应式设计，支持移动端 Web 访问
- **可维护性**：前后端分离，Swagger 文档自动生成，数据库迁移脚本化

## 3. 功能模块

### 3.1 用户端模块
| 模块 | 功能 | 关键特性 |
|------|------|----------|
| 用户认证 | 注册、登录、令牌刷新 | JWT（访问令牌+刷新令牌） |
| 联系人管理 | 基础信息、国家、自定义字段 | 多条件筛选、批量操作 |
| 分组与标签 | 树形分组，多对多标签 | 支持从系统库导入 |
| 互动日志 | 记录沟通时间、类型、摘要 | 时间轴展示，筛选统计 |
| 提醒中心 | 周期性提醒，联系预警 | 支持农历生日（扩展） |
| 亲属关系 | 系别、辈分、称呼 | 关系网络，家族树 |
| 绑定协作 | 发起绑定、资料同步 | 仅同步基础字段 |
| 通知 | 系统内消息列表 | 未读角标，消息分类 |
| 国际化 | 中英文界面切换 | 语言本地存储+同步用户偏好 |

### 3.2 管理端模块
| 模块 | 功能 | 说明 |
|------|------|------|
| 用户管理 | 查看所有用户，禁用/启用/删除 | 显示联系人数量、最近活跃 |
| 绑定审核 | 查看所有绑定申请，强制处理 | 管理员可干预异常申请 |
| 通用标签库 | 预设标签CRUD，多语言名称 | 用户可一键导入到私有标签 |
| 分组模板 | 预设分组模板CRUD，多语言名称 | 用户可导入为私有分组 |
| 系统公告 | 创建多语言公告 | 全站用户可见 |
| 统计面板 | 用户数、活跃度、绑定对数 | 图表展示 |

## 4. 技术选型与架构

### 4.1 总体架构
```
[前端] Vue 3 SPA (响应式)
  |
  | HTTPS /api/v1
  |
[后端] FastAPI (异步)
  |--- 中间件: CORS, JWT Auth, 语言检测
  |--- ORM: SQLAlchemy (async)
  |--- 数据库: SQLite / MySQL / PostgreSQL
         |--- Alembic 迁移管理
```

### 4.2 前端技术栈
- **框架**：Vue 3 + Composition API
- **构建工具**：Vite
- **状态管理**：Pinia
- **路由**：Vue Router 4 (含导航守卫)
- **UI 组件库**：Element Plus
- **国际化**：vue-i18n
- **图表**：ECharts (管理统计)
- **可视化**：d3.js 或 vue-flow (家族树)
- **HTTP**：axios
- **其他**：dayjs, flag-icons

### 4.3 后端技术栈
- **框架**：FastAPI (async)
- **数据验证**：Pydantic V2
- **ORM**：SQLAlchemy 2.0 (async mode)
- **数据库驱动**：aiosqlite, asyncpg, aiomysql
- **迁移**：Alembic
- **认证**：python-jose (JWT), passlib (bcrypt)
- **文档**：Swagger UI (自动生成)
- **国际化**：自定义消息中间件（或 gettext）
